# Fit term - 要件定義書

## 概要

Fit term は iOS 向けの SSH クライアントアプリである。既存の SSH クライアントアプリでも同様の機能を持つものは存在するが、本アプリは**キーボード拡張エリアのカスタマイズ性**に重点を置き、ユーザーが自分の使い方に合わせて自由にレイアウトを構築できることを最大の特徴とする。

## なぜ作るのか

iPhone の純正キーボードには、SSH 操作に不可欠な以下のキーが存在しない:

- Command / Control / Alt (Meta) キー
- Escape キー
- 矢印キー (↑↓←→)
- Tab キー
- Fn キー

既存の SSH クライアントアプリはこれらの補助キーを提供しているが、ボタンの配置やサイズは開発者が決めた固定レイアウトに従うものがほとんどである。利き手、指の長さ、よく使うキーの組み合わせは人によって異なるにもかかわらず、ユーザーがそれを自由に変更できるアプリは少ない。

Fit term は「ユーザーが自分専用のキーボード拡張エリアを自由に設計できる」ことをコアバリューとして開発する。

## 機能要件

### 1. SSH 接続基本機能

- ホスト、ポート、ユーザー名、認証方式（パスワード）を指定して接続
- ターミナルエミュレーション（xterm-256color 等）
- **ターミナル表示設定**: フォント種類・フォントサイズ・背景色をユーザーが変更可能
- **複数セッションのタブ管理**: 複数のサーバーに同時接続し、タブで切り替えて操作できる。同時接続数に上限なし
- **接続プロファイル**: よく接続するサーバーの情報（ホスト・ポート・ユーザー名・認証情報）を保存し、一覧から選んで接続できる
- **認証方式**: パスワード認証

### 2. カスタマイズ可能なキーボード拡張エリア

**本アプリの最大の差別化ポイント。**

純正キーボードの上部に、ユーザーが自由にボタンを配置できる拡張エリアを設ける。

#### 配置できる要素

- **キーアクションボタン**: 単一キー（Escape, Tab, 矢印キーなど）またはキーの組み合わせ（Ctrl+C, Shift+Tab, Ctrl+D など）をワンタップで送信するボタン。修飾キー単体のトグル式は設けず、組み合わせ済みのワンショットボタンとして登録する
- **スニペットボタン**: ユーザーが登録した任意のコマンド文字列を送信するボタン（後述）

#### ボタンの見た目カスタマイズ

- **ラベル**: ユーザーが任意の表示テキストを設定できる（例: 「Shift+Tab」「S+T」「⇧⇥」など自由に決められる）
- **色**: ボタンの色をユーザーが変更できる

#### レイアウト方式

- **グリッドベース**: ボタンはグリッドに沿って配置する（フリーレイアウトではない）
- **レイアウトは1つ**: 接続先ごとの切り替えは不要。全セッションで共通の1レイアウトを使用する
- **エリアの高さ可変**: ユーザーが拡張エリアの段数（行数）を自分で調整できる（例: 1段 / 2段 / 3段）。ボタンを多く配置したい人は段数を増やし、最小限でよい人は1段にする

#### カスタマイズ項目

- **位置**: グリッド上の任意のセルにボタンを配置できる
- **サイズ**: ボタンの大きさを個別に変更できる（複数セルにまたがるボタンも可）
- **並び順**: ドラッグ＆ドロップ等でボタンの並びを自由に変更できる
- **グルーピング**: 関連するボタンをまとめて配置できる（例: 矢印キー4つを十字配列に並べる）

#### ユースケース例

| ユーザー | カスタマイズ内容 |
|---|---|
| 右利きのユーザー | 矢印キーを右側に集約配置 |
| Vim ユーザー | Escape キーを大きく目立つ位置に配置 |
| サーバー管理者 | Ctrl+C, Ctrl+D をワンタップボタンとして配置し、赤色で目立たせる |
| 開発者 | よく使うスニペットを中央に大きく配置 |

### 3. スニペット機能

ユーザーが任意のコマンド文字列を「スニペット」として登録し、ワンタップで SSH セッションに送信できる。

#### 機能詳細

- スニペットの登録数に制限なし
- 各スニペットにはラベル（表示名）とコマンド文字列を設定
- 送信時に末尾に改行を付与するかどうかを選択可能
- スニペットの登録・編集は**専用の設定画面**で行う（キーボード拡張エリアの編集モードとは分離）
- 登録済みスニペットをキーボード拡張エリアにボタンとして配置する（セクション 2 と統合）

#### スニペット例

| ラベル | コマンド |
|---|---|
| Claude Code | `claude\n` |
| Home | `cd ~\n` |
| Projects | `cd ~/dev/projects\n` |
| Git Status | `git status\n` |
| Docker PS | `docker ps\n` |

### 4. バックグラウンド接続維持

**必須要件。**

アプリがバックグラウンドに移行しても SSH 接続を維持する。ユーザーがホーム画面に戻ったり、他のアプリを使用した後に Fit term に復帰した際、再接続なしでそのままセッションを継続できること。

#### 技術的考慮事項

- iOS の Background Modes（例: VoIP, Background Processing）の活用
- Keep-alive パケットの定期送信
- 接続断を検知した場合の自動再接続機構
- ユーザーへの接続状態のフィードバック（接続中 / 再接続中 / 切断）

## 画面構成

### 起動画面（ホーム）

アプリ起動時は**接続プロファイル一覧**を表示する。ユーザーはここから保存済みのサーバーを選択して接続するか、新規接続を追加する。

#### 接続プロファイル

- 各プロファイルにはユーザーが自由に決められる**ニックネーム**を設定（例:「自宅サーバー」「会社VPS」）
- 保存項目: ニックネーム、ホスト（IP）、ポート、ユーザー名、認証方式・認証情報
- **プロファイルの複製機能**: 既存のプロファイルから設定を複製して新規プロファイルを作成できる。同一サーバーに対してユーザー名や認証方式を変えた複数プロファイルを効率的に作成するため

### ターミナル画面

SSH 接続中のメイン画面。ターミナル表示＋キーボード拡張エリアで構成される。複数セッションはタブで切り替える。

### 設定画面

以下の設定項目を**1つの設定画面に統合**する:

- **キーボード拡張エリアのレイアウト編集**: グリッド上のボタン配置・段数の変更など。編集は設定画面からのみ行う（ターミナル画面上での長押し編集モード等は設けない）
- **スニペット管理**: スニペットの追加・編集・削除
- **ターミナル表示設定**: フォント種類・フォントサイズ・背景色の変更

---

## 非機能要件

- iPhone のみ対応（iPad は対象外）
- iOS 17 以上をサポート
- SwiftUI ベースの UI 構築
- SSH ライブラリは Swift 対応のものを使用（NIOSSHなど）
