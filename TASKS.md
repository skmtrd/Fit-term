# CC term - 実装タスク分割

## 技術決定事項

| 項目 | 決定 |
|---|---|
| ターミナルエミュレーション | SwiftTerm ライブラリを使用 |
| データ永続化 | SwiftData |
| バックグラウンド接続維持 | 後のフェーズで検討 |
| 鍵認証 | アプリ内生成 + ファイルインポート両方 |
| SSH ライブラリ | Citadel（導入済み） |
| UI フレームワーク | SwiftUI |
| 対応 OS | iOS 17+ |

---

## Phase 1: コア SSH & ターミナルエミュレーション

現在のコマンド実行型（`executeCommand`）から、インタラクティブシェル（PTY）+ SwiftTerm による本格的なターミナルエミュレーションへ移行する。

### 1-1. SwiftTerm の導入
- [ ] SPM で SwiftTerm パッケージを追加（`https://github.com/migueldeicaza/SwiftTerm.git`）
- [ ] SwiftTerm の `TerminalView`（UIKit）を `UIViewRepresentable` で SwiftUI にラップ

### 1-2. インタラクティブシェルの実装
- [ ] `SSHService` に `startShell()` メソッドを追加（Citadel の PTY API を使用）
- [ ] PTY パラメータ: `xterm-256color`、端末サイズ（cols/rows）を SwiftTerm から取得
- [ ] SSH サーバーからの出力を SwiftTerm の `feed()` に渡すストリーム処理
- [ ] SwiftTerm からのユーザー入力を SSH チャンネルに送信するコールバック
- [ ] 端末リサイズイベントの送信（SwiftTerm のリサイズ → SSH `windowChange`）

### 1-3. TerminalView の刷新
- [ ] 現在のテキストベース `TerminalView` を SwiftTerm ベースに置き換え
- [ ] SwiftTerm の `TerminalViewDelegate` を実装（入力送信、サイズ変更通知）
- [ ] 既存の `TerminalViewModel` をインタラクティブシェル対応に書き換え
- [ ] 接続 → シェル起動 → ターミナル表示の一連フロー完成

### 1-4. 基本的な接続フローの改善
- [ ] 接続中のローディング表示
- [ ] 接続エラー時のリトライ UI
- [ ] 切断検知とユーザーへのフィードバック

## 実装順序と依存関係

```
Phase 1 (SSH & ターミナル)
  └→ Phase 2 (プロファイル & 永続化)
       ├→ Phase 3 (鍵認証)
       └→ Phase 4 (タブ管理)
            └→ Phase 5 (キーボード拡張) ←── Phase 6 (スニペット)
                 └→ Phase 7 (表示設定)
                      └→ Phase 8 (設定画面統合)
                           └→ Phase 9 (バックグラウンド)
```

**推奨実装順**: Phase 1 → 2 → 3 → 4 → 6 → 5 → 7 → 8 → 9

> Phase 6（スニペット）を Phase 5（キーボード拡張）より先にするのは、
> スニペットのデータモデルが先にないとキーボード拡張のボタンにスニペットを割り当てられないため。
