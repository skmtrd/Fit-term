# CC term - 実装タスク分割

## 技術決定事項

| 項目 | 決定 |
|---|---|
| ターミナルエミュレーション | SwiftTerm ライブラリを使用 |
| データ永続化 | SwiftData |
| バックグラウンド接続維持 | 後のフェーズで検討 |
| 認証方式 | パスワード認証のみ |
| SSH ライブラリ | Citadel（導入済み） |
| UI フレームワーク | SwiftUI |
| 対応 OS | iOS 17+ |

---

## Phase 1: コア SSH & ターミナルエミュレーション

現在のコマンド実行型（`executeCommand`）から、インタラクティブシェル（PTY）+ SwiftTerm による本格的なターミナルエミュレーションへ移行する。

### 1-1. SwiftTerm の導入
- [x] SPM で SwiftTerm パッケージを追加（`https://github.com/migueldeicaza/SwiftTerm.git`）
- [x] SwiftTerm の `TerminalView`（UIKit）を `UIViewRepresentable` で SwiftUI にラップ

### 1-2. インタラクティブシェルの実装
- [x] `SSHService` に `startShell()` メソッドを追加（Citadel の PTY API を使用）
- [x] PTY パラメータ: `xterm-256color`、端末サイズ（cols/rows）を SwiftTerm から取得
- [x] SSH サーバーからの出力を SwiftTerm の `feed()` に渡すストリーム処理
- [x] SwiftTerm からのユーザー入力を SSH チャンネルに送信するコールバック
- [x] 端末リサイズイベントの送信（SwiftTerm のリサイズ → SSH `windowChange`）

### 1-3. TerminalView の刷新
- [x] 現在のテキストベース `TerminalView` を SwiftTerm ベースに置き換え
- [x] SwiftTerm の `TerminalViewDelegate` を実装（入力送信、サイズ変更通知）
- [x] 既存の `TerminalViewModel` をインタラクティブシェル対応に書き換え
- [x] 接続 → シェル起動 → ターミナル表示の一連フロー完成

### 1-4. 基本的な接続フローの改善
- [x] 接続中のローディング表示
- [ ] 接続エラー時のリトライ UI
- [x] 切断検知とユーザーへのフィードバック
- [x] キーボード表示時のターミナル自動リサイズ

## 実装順序と依存関係

```
Phase 1 (SSH & ターミナル)
  └→ Phase 2 (プロファイル & 永続化)
       └→ Phase 3 (タブ管理)
            └→ Phase 4 (キーボード拡張) ←── Phase 5 (スニペット)
                 └→ Phase 6 (表示設定)
                      └→ Phase 7 (設定画面統合)
                           └→ Phase 8 (バックグラウンド)
```

**推奨実装順**: Phase 1 → 2 → 3 → 5 → 4 → 6 → 7 → 8

> Phase 5（スニペット）を Phase 4（キーボード拡張）より先にするのは、
> スニペットのデータモデルが先にないとキーボード拡張のボタンにスニペットを割り当てられないため。
